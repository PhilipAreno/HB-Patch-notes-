COMING SOON Schedule Builder Agent (SBA) — Version 1 Jul 27, 2026
What Changed
A new AI-powered scheduling mode inside Schedule Builder on OAM Web. When a manager opens an empty week, they're prompted to let the agent build a draft. It generates shifts based on roles and recent patterns, then surfaces conflicts in a sidebar (pending PTO, overtime, clopening, no coverage, stale availability) to resolve before publishing. Fully opt-in — standard Schedule Builder is unchanged. Mobile not in V1. Phased rollout via flag schedule_builder_agent; full target ~Aug 3.
Audience
All OAM Web users. Phased rollout starting with locations that have schedule stability ≥70%, availability data for ≥50% of team, and ≥5 scheduled employees.
CS Guidance
Direct managers to Schedule Builder → open any unpublished empty week → accept "Let the agent build this." Dismiss with "Not now" to return to manual mode (paused 7 days). If the draft looks off — expected in V1, it's a starting point. If the canvas looks unfamiliar, confirm they opted in; standard view is always available.
IMPROVEMENT Missed Break Corrections — Skip Optional Meal Breaks Jun 29, 2026
What Changed
A new sub-toggle — "Skip optional meal breaks" — is now available under Settings → Breaks & Compliance → Break settings → Missed break corrections. Default: OFF (no behavior change). When ON, clock-out only prompts about mandatory breaks; optional meal breaks are not flagged. Gated behind rollout flag mbc_mandatory_only.
Audience
OAMs at locations with Missed Break Corrections enabled (~536 locations). Employees only see reduced prompts when an OAM opts in.
CS Guidance
Clock-out checklist too long? Settings → Breaks & Compliance → Break settings → Missed break corrections → enable "Skip optional meal breaks." Sub-toggle not visible? Confirm MBC is on and location is on the mbc_mandatory_only rollout.
IMPROVEMENT Employee Termination Flow — Two Updates Jun 25, 2026
What Changed
1. New termination reason: "Project Completed" — now in the reason dropdown (employee profile + bulk terminate). For planned end-of-gig/show/project employment.

2. "Eligible for re-hire" checked by default — pre-selected on the termination flow. ~70% of terminated employees are rehire-eligible, so this is the common case.
Audience
All customers terminating employees. No rollout flags — both are additive UX changes.
CS Guidance
If a customer asks for a custom termination reason, point them to the dropdown first — we add standard options, not free-form text. The rehire default requires no customer action; managers can uncheck it when needed.

IMPROVEMENT Payroll Company Archival — Active Payroll Block Jun 22, 2026
What Changed
CS reps can no longer archive companies in ActiveAdmin that have an active payroll subscription. The system now blocks the action to prevent accounts entering a bad state.
Audience
CS reps working in ActiveAdmin. Internal workflow change — no customer-facing impact.
CS Guidance
CS workflow change: Before archiving any company, confirm the payroll subscription is canceled first. Steps: cancel payroll subscription → then archive the company.
BUG FIX PTO Requests from Terminated Employees — Fixed Jun 18, 2026
What Changed
Managers will no longer receive any notifications (email, SMS, push, or in-app) for time-off requests from terminated employees. Stale requests are also automatically removed from the Requests Feed inbox.
Audience
All managers who receive time-off request notifications.
CS Guidance
If a manager reports receiving PTO notifications from former employees, this fix is live. Requests from terminated employees clear automatically — no action needed from the customer.

IMPROVEMENT New Homebase Left Nav — Now Live for All Customers Jun 17, 2026
What Changed
The redesigned left navigation is now at 100%. Key layout changes: Schedule → Shifts (time off + availability live here), Pay consolidates payroll and tip pooling, HR groups HR Pro/Benefits/Documents, Ops brings together reports/training/Task Manager/Manager Log. Settings and location picker moved to top-left. Trial status bar visible in nav. Payroll Summaries page removed (redundant with Pay Period review).
Audience
All Homebase customers. Rolled out in phases since May 11; 100% as of June 17.
CS Guidance
Every customer is now on the new nav. Any "I can't find X" call is almost certainly a navigation question. Use the new layout names when directing customers — "Schedule" is now "Shifts," "Timesheets" stays, but payroll summary is under Pay Period view → "View summary."
IMPROVEMENT Timesheets — View Summary Navigation Redesign Jun 16, 2026
What Changed
The payroll summary is now a dedicated navigable page. Non-payroll OAMs: "View summary" navigates directly from the Pay Period view. Payroll OAMs: Tools menu → "View summary" or "Download time cards." Clover OAMs: "Send to ADP" opens the summary page instead of a modal. All account types get a date picker and breadcrumb. The Download Timesheets modal has been removed.
Audience
All OAMs on Timesheets Pay Period view.
CS Guidance
Direct customers to Timesheets → Pay Period view → "View summary." Payroll: Tools → "View summary." CSV: Tools → "Download time cards." Clover: "Send to ADP" opens the summary page. The old Download modal is gone — don't reference it.
COMPLIANCE California Alternative Workweek (4×10) OT Support Jun 15, 2026
What Changed
CA employers can now enable "Team member works 4 days, 10-hour shifts, per week" on an employee's job profile. Daily OT kicks in after 10 hours (not 8). Location: Team → Employee Profile → Job Details → Access & Wages. Opt-in per employee. CA double OT (12+ hrs) and 7th-day OT rules are unchanged.
Audience
CA customers with employees on an approved 4×10 alternative workweek schedule. Rollout flag payroll_california_alternative_workweek at 100%.
CS Guidance
Opt-in only — must be enabled per employee after the employer completes CA's alternative workweek election. If a customer asks about 4×10 OT, direct them to Team → Employee Profile → Job Details → Access & Wages.

IMPROVEMENT Mobile Dashboard — New Hire Documents Card Redesign May 11, 2026
What Changed
The New Hire Packet (NHP) card is now pinned to the top of the mobile home screen. Card shows 3 states (Start / In Progress / Overdue), document completion count (e.g., "4 of 7"), days until start date, and overdue status when past the employer's deadline. Card disappears once all items are completed. ~13,500 packets sent monthly.
Audience
New employees with an active New Hire Packet. View logic (when the card appears/disappears) is unchanged — only the placement and context have changed.
CS Guidance
No changes to how NHP works. If employees report issues accessing documents, report to the team. The card should be the most prominent item on the home screen for any new hire with an incomplete packet.
IMPROVEMENT New Hire Packet — Web Dashboard + Email Redirect Fix May 7, 2026
What Changed
Four fixes to the NHP completion funnel (completion rate had dropped below 40%): (1) NHP now surfaces on the web employee dashboard; (2) NHP emails redirect directly to the packet after sign-in instead of a login wall dead-end; (3) garbled characters in the email subject line fixed; (4) email-to-completion funnel re-instrumented in Amplitude. Rollout flag: nhp_employee_access_web.
Audience
~13,500 NHP packets sent per month. Large subset of this traffic now expected to go through web.
CS Guidance
Employees who previously hit a dead-end login wall from NHP emails can now complete their packet directly on web. If an employee reports not seeing their NHP on the web dashboard, check if they're in the nhp_employee_access_web rollout.
IMPROVEMENT Break Settings — Full Overhaul May 6, 2026
What Changed
Major overhaul of the Breaks experience: Break Settings page reorganized with clearer groupings and copy; break policies can now be archived to reduce clutter; employees can enter missed breaks at clock-out even without break waivers enabled; missed break corrections decoupled from break waivers; state-specific availability of break penalties improved; default optional break behavior made consistent.
Audience
All customers that use Homebase to track mandatory and optional breaks.
CS Guidance
Be ready to walk customers through the reorganized Break Settings page. Employees can now enter missed breaks at clock-out independently of whether the break waiver setting is enabled — this is intentional, not a bug.
